# External audit: collections.ts + export.ts

Target: `src/handlers/collections.ts` and `src/handlers/export.ts` (Zotero 8.0.4).
Row format and verdict set: see `2026-04-23-xpi-api-audit-design.md`.

Scope note: this table may include calls on Zotero-returned instances that the
narrow `Zotero\.\w+\.\w+|new Zotero\.\w+` regex does not pick up (e.g.
`col.save()`, `translate.setItems()`). When such a call carries a finding, add
it as its own row keyed on its real line number ‚Äî the row total will exceed the
bare grep count. Declare the actual total in this note after Step 3.

**Actual enumeration totals declared here after Step 3:**

- `collections.ts`: narrow-grep finds **16** `Zotero.*` call sites. No extra
  instance-method findings were severe enough to warrant split rows ‚Äî every
  `col.saveTx()`, `col.eraseTx()`, `col.getChildItems()`, `col.getChildCollections()`,
  `item.addToCollection()`, `item.removeFromCollection()`, `item.saveTx()` call
  uses the documented signature correctly. Table therefore has **16 rows**. The
  task plan said "expected 17" ‚Äî that was a plan-side miscount; the file truly
  has 16 matches (see Step 1 output in commit message).
- `export.ts`: narrow-grep finds **4** `Zotero.*` call sites (one of which is
  the comment on L48 referring to `Zotero.QuickCopy.getContentFromItems`, which
  is no longer called ‚Äî retained as an audit row explaining the bypass). The
  `bibliography` handler additionally performs several Zotero-instance calls
  (`Zotero.Styles.get`, `style.getCiteProc`, `engine.setOutputFormat`,
  `engine.updateItems`, `engine.makeBibliography`, `engine.free`) that the
  narrow regex misses ‚Äî these are the load-bearing API surface of the 2026-04-23
  bibliography rewrite and MUST be audited. Each gets its own row. Similar
  treatment for the `exportItems` helper's instance methods
  (`translate.setItems`, `translate.setTranslator`, `translate.setHandler`,
  `translate.translate`, `translate.string`) and the `citationKey` handler's
  `item.getField` / `item.getCreators`. Table therefore has **18 rows** total
  (4 grep-matched + 14 instance-method / property extras). The task plan said
  "expected 5" ‚Äî again a plan-side miscount plus the deliberate scope widening
  for the bibliography rewrite's API surface.

Total rows across both sections: **34**. Phase 6's concatenation uses a broader
match that covers all of them.
