# External audit: attachments.ts + notes.ts + search.ts

Target: three handlers, Zotero 8.0.4. Row format / verdicts per design doc
(`2026-04-23-xpi-api-audit-design.md`).

Scope note: this table may include calls on Zotero-returned instances that the
narrow `Zotero\.\w+\.\w+|new Zotero\.\w+` regex does not pick up. Such calls
get their own rows keyed on real line numbers when they carry findings, so row
totals may exceed narrow-grep counts.

**Actual enumeration totals declared here after Step 3:**

- `attachments.ts`: narrow-grep finds **15** `Zotero.*` call sites (the task
  plan said "expected 16" — plan-side miscount; confirmed `grep -cE ... =
  15`). All 15 are audited. Instance-method calls
  (`parent.getAttachments`, `att.getField`, `att.getFilePath`,
  `att.getFilePathAsync`, `item.isAttachment`, `file.exists`) are correct
  against item.js and file.js, no split rows needed. Table has **15 rows**.
- `notes.ts`: narrow-grep finds **14** `Zotero.*` call sites (matches plan).
  Two extra rows split out: `s.addCondition("note", "contains", query)` on
  line 48 (load-bearing — verifies the `note` condition name + operator), and
  `pdfItem.getAnnotations()` on line 71 (the cast `as unknown as number[]` is
  a lie — see finding). Table has **16 rows**.
- `search.ts`: narrow-grep finds **19** `Zotero.*` call sites (plan said "20";
  plan-side miscount). Table has **23 rows** — four extra rows split out for
  instance-method calls that the narrow regex misses but whose correctness is
  load-bearing for every search method: `s.addCondition(...)` (search.ts:11,
  12) and `s.search()` (search.ts:13) are all audited against search.js#L297
  / search.js#L541; the `addCondition("noChildren", ...)` pattern gets one
  shared row at search.ts:12 — the same idiom appears 5 times (L12/L29/L44/
  L60/L77) and the fix is a single `replace_all`. Note: `search.byIdentifier`
  still types `params.pmid?: string` in its signature (line 70) even though
  commit `b7b5e21` removed `--pmid` from the CLI — no handler branch uses it.
  Already catalogued in prior tasks; not re-flagged as its own row.

Total rows across three sections: **54** (15 + 16 + 23). Grep total is **48**
(15 + 14 + 19). The +6 widening is from instance-method splits (2 in notes.ts,
4 in search.ts) that carry findings or verify load-bearing signatures.
